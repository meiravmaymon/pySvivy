DATABASE SCHEMA:
================

1. PERSONS TABLE
   - Personal information (id, full_name, title, role, faction)
   - Academic degree
   - Term dates (start_date, end_date)
   - Gender
   - Absence counter
   - Relationships: Many-to-Many with Boards, One-to-Many with Votes/Attendances

2. BOARDS TABLE (Committees)
   - Committee information (id, title, committee_type)
   - Term dates
   - Description
   - Relationships: Many-to-Many with Persons, One-to-Many with Meetings

3. MEETINGS TABLE (Protocols)
   - Meeting details (id, title, meeting_no, meeting_date)
   - Protocol file link (PDF)
   - Relationships: Many-to-One with Board, One-to-Many with Discussions/Attendances

4. DISCUSSIONS TABLE (Sections/Issues)
   - Discussion content (id, title, discussion_type, categories)
   - Issue number
   - Expert opinion and decision text
   - Budget information
   - Vote counters (yes, no, avoid, missing)
   - Relationships: Many-to-One with Meeting, One-to-Many with Votes

5. VOTES TABLE
   - Individual vote records (person_id, discussion_id, vote)
   - Vote values: 'yes', 'no', 'avoid', 'missing'
   - Relationships: Many-to-One with Person and Discussion

6. ATTENDANCES TABLE
   - Meeting attendance records (person_id, meeting_id, is_present)
   - Relationships: Many-to-One with Person and Meeting

7. PERSON_BOARD TABLE (Many-to-Many junction)
   - Links persons to boards with term dates
